* Зачем нужен GIT и какие задачи решают с его помощью?

Git -  система контроля версиями (хранит всю историю изменений). Он решает 
задачи:
1. Защиты исходного кода от потери (если репозиторий размещен на удаленном 
сервере)
2. Обеспечения командной работы (одним репозиторием параллельно пользуются 
несколько человек)
3. Отслеживания изменений


* Что такое репозиторий GIT и где он находится?

Git репозиторий - это все файлы с историей, находящиеся под контролем 
версий.
Он находится либо на локальном компьютере (локальная папка .git), либо на 
каком-то сетевом ресурсе (к примеру, GitHub).  


* Для чего используется файл .gitignore?

.gitignore - файл, в который помещаются маски файлов, которые по 
какой-либо причине необходимо игнорировать, чтобы те не попадали в 
коммиты.


* Что такое "коммит"?

Набор изменений и метаданных, сохраненных в дереве версий (снимок 
репозитория).
Операция commit - «фиксировать, сохранить изменения» - операция сохранения 
и подтверждения внесенных изменений в репозиторий.



Укажите базовые команды в рамках работы с GIT:

* Настройка аккаунта локального пользователя.

	git config --global user.name "John Doe"
	git config --global user.email johndoe@example.com


* Создание репозитория.

 	git init 


* Добавление файлов в индекс и выполнение коммита.

	git add file_name file_name
	git add .   # рекурсивно добавит все файлы


* Просмотр журнала коммитов.

	git status
