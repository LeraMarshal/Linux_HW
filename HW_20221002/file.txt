1. Коротко опишите, какие задачи с помощью механизма ветвления решаются в GIT? Зачем выполняется их 
слияние?

	Механизм ветвления в GIT используется для разделения кода, который можно легко откатывать, если 
вдруг мы передумаем его сливать в основную ветку, либо делать несколько различных изменений в разных 
ветках. Также ветке нужны для того, чтобы программисты могли вести совместную работу над проектом и не 
мешать друг другу при этом.

Слияние веток выполняется для того, чтобы переместить изменение из одной ветки в другую.



2. Укажите команды для:


* переименования веток ->
    1. Удалить старую_ветвь на GH
    2. Сделать push новой ветви на GH:
    3. git branch -m новое_название
    4. git origin --delete старое_название
    5. git push -u origin новое_название 


* создания и переключения на новую ветку ->
    1. git branch имя_новой_ветки
    2. git checkout (переключение веток)


* удаление ветки ->
	git branch -d название_ветки


* слияния веток ->
	git merge название_ветки
